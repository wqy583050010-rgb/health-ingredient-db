# 数据更新说明（模板 + 导入）

网站的数据（原料、代工厂）现在用**文本模板**来管理。不用懂代码，照着模板填就行。

## 一、两种模板文件

| 文件 | 用途 |
|------|------|
| `data/TEMPLATE_原料.txt` | 添加 / 修改一种原料 |
| `data/TEMPLATE_代工厂.txt` | 添加 / 修改一家代工厂 |

每个模板里都有详细注释，告诉你每个字段怎么填。复制模板里 `=== 原料 ===`（或 `=== 代工厂 ===`）到结尾的那一段，填上值即可。

### 填写规则（很简单）
- 每行：`字段: 值`
- 多个值用「/」分开，例如：`形式: 胶囊 / 片剂 / 粉剂`
- 供应商、合规、机理步骤、产品案例：用「|」分开每一项，多条用「/」分开
- 以「#」开头的行是注释，不导入
- 不会填的留空，系统自动补空白，不会报错
- **想改已有的条目**：把「名称」写成和库里一模一样，导入时会自动更新那一条（只覆盖你填的字段，其它保持不变）

## 二、日常怎么用（推荐方式）

你不用自己操作电脑，流程是：

1. 你告诉我："要加一种原料：XXX" 或 "加一个代工厂：YYY"，并把你知道的信息发给我。
2. 我按模板生成一份文本文件，发给你确认。
3. 你看一眼，哪里要改自己改（或让我改）。
4. 我把文本"导入"数据库 → 重新构建 → 发布到线上，几分钟网页就更新了。

> 以后你随时过来，一句话就能加数据，不用碰代码、不用登录任何后台。

## 三、如果你想自己导入（可选，需本机有 Git + Node）

```bash
# 1) 把填好的文本保存成 import.txt，放在项目目录下
# 2) 预览（不写入，先看看对不对）
python scripts/import_text.py import.txt --dry-run

# 3) 确认无误，真正导入
python scripts/import_text.py import.txt

# 4) 重新构建并发布
python scripts/build_from_json.py
npm run build
git add -A
git commit -m "更新数据"
git push
```

GitHub Actions 会自动把新内容发布到网站。

## 四、常见问题

- **导入后网页没变？** 构建发布需要 1~2 分钟；如果还看不到，浏览器按 `Ctrl + F5` 强制刷新。
- **名称填错会怎样？** 如果"名称"和库里任何一个都不一样，会当成**新增**；如果想改旧的，名称必须完全一致（含空格）。
- **字段填错了能改吗？** 能。重新填一份模板，名称写成一样，再导入一次就会覆盖更新。
- **在线管理面板（admin.html）还需要吗？** 不需要。那个面板要登录 GitHub 令牌，比较麻烦，已经弃用；现在统一用"文本模板 + 导入"这种方式。
